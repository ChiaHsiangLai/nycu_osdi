# linker.ld 筆記

對應 [linker.ld](linker.ld) 的說明文件。

## 背景

本質上是一張「記憶體佈局藍圖」：告訴連結器把哪些東西依序放在哪個位址。
核心機制是位置指標「.」：

- 可以手動設定它：`. = 0x80000;`
- 放東西進去之後，它會自動往前跳（跳過剛放進去的內容大小）
- 可以讀出它目前的值，存成一個 symbol：`__bss_start = .;`

產生的記憶體佈局大致長這樣：

```
0x80000  +-------------+
         |   .text     |  程式碼（機器指令）
         +-------------+
         |   .rodata   |  唯讀資料（字串常數等）
         +-------------+
         |   .data     |  有初始值的全域變數
         +-------------+ <- 補齊到 8 的倍數 (ALIGN(8))
__bss_start ---------+
         |   .bss      |  沒初始值的全域變數（開機後要被 boot.S 清成 0）
__bss_end -----------+
```

注意：`.text`/`.rodata`/`.data`/`.bss` 這些名字不是 linker script 語言的保留字，
是編譯器/組譯器自動幫輸入內容取的慣例名稱（可以用 `objdump -h a.o` 驗證），
我們在這裡沿用同樣的名字純粹是為了可讀性、跟其他工具（objdump/gdb）相容。

## `.bss` 段落

`*(COMMON)`：舊式 C「暫定宣告」的全域變數也會被歸類到這裡，一併收集才清得到。

## `_stack_top`

stack 的頂端位址。跟程式碼一樣從 0x80000 開始，
但 stack 是往低位址成長，跟程式碼成長方向相反，不會互踩。
